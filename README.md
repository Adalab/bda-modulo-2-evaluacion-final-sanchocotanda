# bda-modulo-2-evaluacion-final-sanchocotanda

Autora: María Sancho

Proyecto basado en la extracción, tratamiento y almacenamiento de datos mediante el uso de una API y bases de datos relacionales.  
El trabajo incluye la creación de una base de datos propia y la realización de consultas tanto sobre esta como sobre otra base de datos previamente creada, abordando así el ciclo completo de vida de los datos.

## Funcionalidades del proyecto

El proyecto permite:
- **Obtener** datos externos mediante peticiones a una API.
- **Transformar** los datos obtenidos.
- **Diseñar y crear** las tablas necesarias para su almacenamiento.
- **Insertar** los datos procesados en una base de datos propia.
- **Ejecutar** consultas SQL para obtener información relevante, tanto de la base de datos creada (*peliculas_adalab*) como de otra existente (*sakila*).

## Técnicas y conceptos aplicados

- Instalación y uso de librerías para el tratamiento de datos.
- Llamadas a APIs y manejo de respuestas en formato JSON.
- Manipulación de datos con Pandas: normalización y cambios de formato.
- Creación de bases de datos y definición de claves primarias y foráneas.
- Consultas SQL utilizando:
  - `SELECT`
  - `WHERE`
  - `LIKE`
  - `JOIN` (INNER, LEFT)
  - `GROUP BY`
  - `ORDER BY`
  - Funciones de agregación: `COUNT`, `AVG`..

## Contenido del repositorio

El repositorio contiene los siguientes documentos:

1. **Ejercicio 1 – Fase 1**: documento `.ipynb` con la instalación y uso de librerías, llamada a la API y tratamiento de datos con Pandas.
2. **Ejercicio 1 – Fases 2, 3 y 4**: documento `.sql` con la creación de la base de datos, inserción de datos y consultas.
3. **Diagrama de la base de datos**: archivo `.png` como apoyo visual para comprender la estructura y las relaciones entre tablas.
4. **Ejercicio 2**: documento `.ipynb` con la conexión y consultas a una base de datos preinstalada.

**Nota:** Este es un proyecto académico, por lo que algunas consultas presentan distintas opciones de resolución con fines de aprendizaje.
